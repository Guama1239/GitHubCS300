Welcome to CS 300, Client-side Web Development!

This course discusses web site design issues and the requirements of e-commerce. Furthermore, it covers the creation of web pages. Hands-on development and group projects are an essential part of this course.

Upon successful completion of this course, you will be able to understand fundamental design principles in building a web page/web site, create web pages using a text editor, understand the pros and cons of web authoring software, and recognize the different types of websites and their distinct purposes as related to the visitors they are intended to attract.

 

Course Objectives:

  • Ability to choose between HTML, CSS and JavaScript as appropriate technique for creating a web page/site
  • Build a website that is visually pleasing and functional
  • Upload files to a server
  • Understand fundamental design principles in building a web page/web site
  • Create .html files that meet the W3C standard
  • Create a logical file system for web documents and supporting files
  • Choose correct design principles for site usability
  • Use CSS and Javascript and/or jQuery for site aesthetics and user interaction
  • Ability to use software tools required to build a website
  • Ability to recognize a site for web presence, visitor use, and e-commerce

Attention! This is a remote and hybrid course.
 
Hybrid courses have some of the traditional face-to-face "seat time," (replaced with Zoom Sessions for Remote Learning) but some of the time has been replaced by online learning activities. The purpose of a hybrid course is to take advantage of the best features of both face-to-face and online learning. A hybrid course is designed to integrate face-to-face and online activities so that they reinforce, complement, and elaborate one another, instead of treating the online component as an add-on or duplicate of what is taught in the classroom. It is essential that you are prepared for each class meeting, thus, all online and reading assignments are to be completed prior to the class in which they are due. You are responsible to print out a screen shot and source code view for all web pages referenced each week, noting how the code renders within the browser view. You may supplement face-to-face class time by visiting during my posted office hours or contacting me via email.
